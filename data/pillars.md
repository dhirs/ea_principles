# Pillars & Focus Areas

Hierarchical map of the catalogue's pillars (P1–P5, mirroring AWS Well-Architected GenAI Lens pillar codes) and their focus areas. Dotted format: `<pillar>.<focus_area>`.

Sustainability — AWS's sixth pillar — is intentionally omitted; its concerns are distributed across Cost Optimization (efficiency), Operational Excellence (lifecycle), and Security (data retention).

---

## 1. Operational Excellence — P1 / GENOPS

> Can the team see what's happening, ship changes safely, and respond when something breaks?

- **1.1** — Model Performance Evaluation — P11 / GENOPS01
- **1.2** — Operational Health & Monitoring — P12 / GENOPS02
- **1.3** — Traceability — P13 / GENOPS03
- **1.4** — Lifecycle Automation — P14 / GENOPS04
- **1.5** — Model Customization — P15 / GENOPS05

## 2. Security — P2 / GENSEC

> Can we prove access is controlled, prompts are protected, and the workload meets enterprise security standards?

- **2.1** — Identity & Access for GenAI — P21
- **2.2** — Data Protection for GenAI — P22
- **2.3** — Control Plane & Access Monitoring — P23
- **2.4** — Prompt Security — P24 / GENSEC04

## 3. Reliability — P3 / GENREL

> When upstream systems or model providers fail, does the workload survive?

- **3.1** — Foundation Model Resilience — P31
- **3.2** — Enterprise Integration Resilience — P32

## 4. Performance Efficiency — P4 / GENPERF

> Does the workload run efficiently at production scale?

- **4.1** — Model Selection & Right-sizing — P41
- **4.2** — State & Throughput Management — P42

## 5. Cost Optimization — P5 / GENCOST

> Are we spending efficiently on tokens, compute, and vendor contracts?

- **5.1** — Inference Cost Optimization — P51
- **5.2** — Vendor Consolidation & Procurement — P52
