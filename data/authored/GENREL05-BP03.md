- 2026-06-27 | GENREL05-BP03 (whole BP — "Verify that agent capabilities are available across all regions of availability"; live page fetched + verified 2026-06-27 from genrel05-bp03.html; mapping_state **verified**) | **not_promoted (user-directed 2026-06-27).** Third BP of GENREL05, AWS risk **Medium**. The BP: an agent's supporting infrastructure (tool APIs / functions) and model access must exist in every Region the agent runs in; deploy agent infra in primary+secondary Regions, route with CloudFront / Route 53 latency-based routing, verify agents can reach required resources per Region, configure model access in all Regions if not using a cross-Region inference profile. **This is the one BP of GENREL05 with a genuinely agent-distinct failure mode** (named at the HALT): regional failover that is transparent for a stateless service can **silently degrade an agent** — if its tool surface or model access is not region-complete in the failover Region, the agent does not error cleanly, it keeps running with a missing capability and produces a worse answer. Base-WAF REL10 does not talk about this; the failover unit is "the agent's whole tool + model surface," not "the service." **Still not_promoted**, because the AWS mechanism (deploy APIs in N Regions + latency routing) is generic infra/landing-zone with no commitable workload-repo artefact, and the agent-distinct part does not (this session) concretise to a CI-gateable check distinct from existing standards: the tool-binding declaration already lives on **GO2B3-01** (side_effecting + idempotency_key per tool binding) and a region-completeness lint would be an extension of that config, not a new standard; routing/topology stays base-WAF. The agent-specific recommendation — *make region-completeness of the agent's tool + model surface an explicit, checkable property* — is worth recording, so unlike BP01/BP02 this BP **gets an ADR** (cf. GENREL01's account-isolation ADR). step_promotion **3/3/3/3 → not_promote** (has_enforceable_artefact 3 — agent-distinct failure correctly named, but no standalone repo artefact survives extraction from infra + GO2B3-01 config; architecturally_distinct 3 — the silent-degradation angle is distinct from base-WAF REL10 yet absorbable as a GO2B3-01 schema extension, named; in_bp_scope 3 — native GENREL05; not_vendor_menu 3 — Bedrock Agents / CloudFront / Route 53 menu quoted).
