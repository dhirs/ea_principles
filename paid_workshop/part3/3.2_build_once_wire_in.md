# Part 3.2 — Build Once, Wire In Many

**Maven one-liner:** Central builds, operates, and owns the machine; projects wire into it through one interface contract — quality lives in the architecture, not in any single architect.

## The point of this subpart

Once prioritization (3.1) decides *what* to build, the next question is *who builds it and who consumes it.* The failure mode is every project rebuilding the same control its own way — the exact sprawl Part 1 diagnosed. The fix is a clean split with one contract between the two sides.

## Central / Platform team — builds, operates, owns

Central builds the control **once, for everyone**, and keeps owning it. Using prompt-caching as the worked example, central owns:

- **The validator** — the engine that checks the control actually holds.
- **The threshold table** — per-model parameters, versioned.
- **The breakpoint emitter** — inserts the control marker at runtime / gateway.
- **The entrypoint contract** — defines the interface projects call.
- **The CI gate** — the check projects plug into; red = blocked.

> Build it once, for everyone.

## Project / Delivery team — configures, populates, consumes

Projects **wire into** the machine; they don't rebuild it:

- Route every call through the **one declared entrypoint.**
- Declare what's in scope vs. out (mark the breakpoint).
- Keep their dynamic/variable content on the right side of the contract.
- Run the central check in CI and fix on red.
- **Don't reinvent the control** — consume the validator, don't hand-roll it.

> Wire into it. Don't rebuild it.

## The contract is the standard

The interface contract — not a memo asking everyone to "remember to do the right thing" — **is** the standard. Because every workload flows through one declared entrypoint, the gate can **prove** the control works instead of trusting that it does. That's what makes the standard enforceable rather than aspirational.

The payoff: **every project is production-ready by default. Quality lives in the architecture — not in the architect.** One published catalogue, enforced at both layers, no heroes required.

## Takeaway for this subpart

> Central builds the machine once and owns it; projects wire in through a single interface contract. The contract is the standard — it lets the gate prove compliance instead of trusting it, so production-readiness becomes a property of the architecture, not of who happened to build the project.
