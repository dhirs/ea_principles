# Part 4.1 — Select the Standard to Build

**Maven one-liner:** Three candidates on the table — one per dimension — and the cohort picks one to build together, using the prioritization lens from Part 3 to justify the choice.

## The point of this subpart

Part 4 is the hands-on lab where everything in Parts 1–3 turns into a built artifact. This first stage is short: it sets up *what* the room will build. Rather than each person building their own standard in parallel (no time, and harder to facilitate), **the whole cohort builds one standard together, live.** This subpart is how we choose that one.

## Show all three dimensions, build one

Put three candidate standards on the table — deliberately one per dimension, so the room *sees* all three even though we only build one:

- **Legal & Risk** candidate
- **Customer Experience** candidate
- **Cost** candidate

This preserves the deck's "three standards, build one" intent: the breadth is in the *exposure* to all three dimensions; the depth is in the *one* we take all the way.

## Pick it with the prioritization lens (callback to 3.1)

Don't pick arbitrarily — use the prioritization framework from sub-part 3.1 as the selection criteria, live:

- **Criticality** — how much risk/value rides on it
- **Difficulty** — how hard the RI is to build
- **Reusability** — how many projects it serves
- **Enforcement cost** — what it costs to gate and operate

Have the room score the three candidates on these and pick the winner. This does double duty: it chooses the build target *and* reinforces Part 3 by applying the gate for real.

## Facilitation note

Keep this tight — a few minutes. The teaching value is in 4.2 (the build). Selection just needs a clear winner everyone is bought into, ideally the one with the strongest criticality × reusability so the worked example feels worth the effort.

## Takeaway for this subpart

> Show all three dimensions, build one. Use the Part 3 prioritization lens to choose, so selection is itself a live application of the gate — not a coin toss.
